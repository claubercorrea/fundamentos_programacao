# 🧮 A Evolução de um Código: Calculadora em Python

Este projeto documenta a minha jornada de aprendizagem no **4º período de Ciência da Computação**, ilustrando a refatoração de uma simples calculadora através de **6 níveis de maturidade técnica**. 

> **Objetivo:** Demonstrar a transição de um script linear para uma arquitetura profissional orientada a objetos (POO).

---

## 🚀 A Jornada de Refatoração

O projeto está dividido em etapas que refletem conceitos fundamentais da **Engenharia de Software**:

### 🔴 [Nível 1] - Lógica Linear e Sequencial
* **Conceito:** Sintaxe básica e operações diretas.
* **Foco:** Identificação de falhas estruturais e falta de validação.

### 🟠 [Nível 2] - Estruturas de Decisão
* **Conceito:** Controle de fluxo com `if/elif/else`.
* **Foco:** Implementação de menus interativos para o usuário.

### 🟡 [Nível 3] - Resiliência e Robustez
* **Conceito:** Tratamento de exceções específicas (`ValueError`, `ZeroDivisionError`).
* **Foco:** Garantir que o software não pare de funcionar perante entradas inválidas.

### 🟢 [Nível 4] - Modularização (DRY)
* **Conceito:** Abstração de procedimentos através de funções.
* **Foco:** Reutilização de código e organização por responsabilidades (**Don't Repeat Yourself**).

### 🔵 [Nível 5] - Validação Avançada e Loops
* **Conceito:** Loops de execução (`while True`) e validação de coleções.
* **Foco:** Melhoria da Experiência do Usuário (UX) contínua.

### 🟣 [Nível 6] - Programação Orientada a Objetos (POO)
* **Conceito:** Classes, Métodos e Encapsulamento.
* **Foco:** Separação total entre a **Lógica de Negócio** (`Classe Calculadora`) e a **Interface do Usuário** (`Classe InterfaceCalculadora`).

---

## 🛠️ Tecnologias Utilizadas

* **Linguagem:** Python 3.x
* **Paradigmas:** Programação Estruturada e Orientada a Objetos (OOP).

## 🧠 Lições Aprendidas (Ciência da Computação)

1.  **Separação de Preocupações (SoC):** A importância de não misturar lógica matemática com entrada/saída de dados.
2.  **Tratamento de Erros:** Antecipar falhas para criar sistemas resilientes.
3.  **Manutenibilidade:** Como um código bem estruturado é mais fácil de ler e atualizar.
